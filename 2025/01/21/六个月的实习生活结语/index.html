六个月的深圳实习生活感悟

​ 六个月前，我通过校招进入一家主攻BigData+AI的产品企业，该企业成为了我出来的第一家实习公司，我进公司之后才发现这个是一家外包公司，于是开始了我的实习生涯，我将会分为六个月讲述，我的工作开展，以此来记录我的生活轨迹。

囧事回忆

刚开始工作时，我以为需要用传统的大数据方式完成任务，于是：

  • 下载了一堆大数据组件，如 Hadoop、Spark 等。
  • 沿用了 学校比赛中的那种思维方式工作，结果一不小心踩了很多坑：
    • 网络问题，堡垒机无法连接
    • 配置 映射规则出错。
    • 各种环境问题接连出现……

回忆起来，这些囧事虽然让当时的我手忙脚乱，但也正因为如此，我和我的师傅、项目经理之间多了很多话题，更深入地了解了彼此。

我的师傅

我的师傅是一位 务实的人，年纪大概 30 多岁出头，有着 十年的工作经验。他的人生经历非常丰富：

  • 曾在 WEB 开发、嵌入式、桌面应用 等领域奉献过青春。
  • 见证过 计算机行业的鼎盛浪潮。
  • 直到 2020 年，他转到了 大数据开发，并持续深耕至今。

带实习生的态度

虽然我可能不是他第一个带的实习生，但他对我 非常认真负责。在他的指导下，我感受到了他的用心：

  • 教我从 底层原理 到 实际应用，一丝不苟。
  • 从不敷衍，耐心解答我的各种问题。

现在回想起来，如果没有那些囧事，我和我的师傅、项目经理可能不会有这么多话题，也不会这么快拉近彼此的距离。那些经历虽显得“狼狈”，却为我留下了 宝贵的成长记忆。

第一个月-萌新小白挑战顶级项目

初入公司：我的第一个项目组经历

我的第一个项目是一次 全过程的离线数据开发，让我对大数据开发有了更全面的认识。此次项目中，使用的是 CDH（Cloudera’s Distribution Including Apache Hadoop），这是一个大数据管理平台，旨在方便开发人员对集群进行管理。

项目概述

本次开发分为两个核心领域：制造域 和 营销域。目标是：

  1. 通过 ETL 加工 对方的数据。
  2. 开发对应的域指标。
  3. 最终将数据加工为 ADS（应用数据层），提供给 BI 系统用于构建 可视化大数据分析大屏。

数据来源与采集

数据主要来自客户的 ERP 系统，如 金蝶，以及 Oracle、MySQL、SQL Server 等关系型数据库。我们将这些数据采集到 Hive 中进行处理，本次项目还引入了 StarRocks，一个高性能分析型数据库，显著加快了我们对数据的查询速度。

数据处理流程

1. 数据 ETL

通过 ETL 流程，我们对原始数据进行了以下操作：

  • 清洗数据：去除脏数据，确保数据质量。
  • 提取维度：针对不同业务需求提取相关维度。
  • 加工指标：根据业务需求对数据进行多轮加工，生成所需指标。

2. HiveSQL 编写

在数据处理阶段，我们主要依赖 SPARKSQL，完成以下任务：

  • 数据清洗与加工。
  • 维度提取。
  • 指标计算与聚合。

这完全取决于 业务需求，这些业务需求由 项目经理 提出，并根据客户讨论的结果确定。而我作为一名 SQL 仔 的职责就是将这些指标实现。

数据仓库设计

此次项目沿用了传统数仓开发的分层架构，主要包括以下四层：

  1. ODS（操作数据层）：存储原始数据。
  2. DIM（维度数据层）：提取与加工业务相关的维度。
  3. DWD（明细数据层）：经过清洗与加工后的明细数据。
  4. DWS（汇总数据层）：针对业务需求进行指标汇总。
  5. ADS（应用数据层）：提供给 BI 系统使用的最终数据。

最后，将 DWS 指标 加工后导入 ADS 表，为 BI 提供统一的数据视图，构建出 大屏可视化分析。

收获

本次项目不仅让我熟悉了 CDH 平台和传统数仓开发流程，也让我深入了解了 ETL 的完整实施过程，让我对大数据有了更深层次的了解。

暂时说到这里，我需要理清楚我的整个过程链路，靠AI帮忙美化文章的效果，有点恼火……
